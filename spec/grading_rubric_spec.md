# Grading Rubric Specification

## Core Rule
**The assignment's own rubric is always the source of truth for marks.** This document is a guideline for how to apply rubrics.

## Grading Approach
- Read the assignment rubric carefully
- Match each rubric criterion against the submission
- Score each criterion based on visible evidence in the code
- Never give full marks unless the submission fully satisfies the criterion
- If a criterion is partially met, give partial marks and explain what's missing
- If evidence is unclear, assume the student hasn't completed it

## Deduction Guidelines (only if in rubric)
- Late submission: as specified in assignment
- Missing files/naming issues: if rubric mentions file naming
- Incomplete output: compare against expected/sample output

## Verdict Guidelines
- 90%+ of total marks: "Excellent work"
- 70-89%: "Good effort"
- 50-69%: "Needs improvement"
- Below 50%: "Requires significant revision"
- Missing major requirements: "Incomplete submission"
